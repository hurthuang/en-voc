# 資料更新與建置流程

記錄「出版社課本改版後，如何重新蒐集/整理資料並更新這個測驗 app」的完整操作流程。
蒐集資料的方法（怎麼從各出版社平台找到單字表）另見 [資料來源與蒐集方法.md](資料來源與蒐集方法.md)；
這份文件講的是「資料改好之後，怎麼重跑管線讓 app 更新」。

---

## 整體資料流

```
英文單字表_*.md（人工整理的來源資料）
        │  node en-test/scripts/build-data.js
        ▼
en-test/*.txt（國中）＋ en-test/elementary/*.txt（國小）
        │  node en-test/scripts/gen-words-list.js
        ▼
en-test/scripts/words.generated.json（全部不重複英文單字清單）
        │  node en-test/scripts/router-server.js （背景執行）
        │  node en-test/scripts/braille-translate-batch.js
        ▼
en-test/scripts/braille-ueb-g2.generated.json（點字碼對照表）
        │  node en-test/scripts/build-data.js （重跑一次，這次會讀到上面的點字碼）
        ▼
en-test/*.txt、en-test/elementary/*.txt 補上點字碼
en-test/scripts/elementary-data.generated.js（國小階層資料）
        │  node en-test/scripts/inject-elementary-data.js
        ▼
index.html 裡的 ELEMENTARY_DATA 陣列同步更新
        │  git add / commit / push
        ▼
GitHub Pages 自動部署（https://hurthuang.github.io/en-voc/）
```

**重點**：`build-data.js` 要跑兩次——第一次是為了把新/改過的單字先落到 txt 檔（此時點字欄位可能是空的或用舊表），第二次是在點字批次翻譯完成、`braille-ueb-g2.generated.json` 更新之後，把正確的點字碼補進 txt 檔。

## 情境一：只是改了某幾課的中文意思／新增幾個單字（沒有新增全新單字）

如果新增的單字在既有的 `braille-ueb-g2.generated.json` 裡已經有點字碼，其實可以跳過整個點字批次翻譯，只要：

```bash
cd en-test
node scripts/build-data.js
node scripts/inject-elementary-data.js   # 注意：是根目錄的 index.html，腳本已處理路徑
```

`build-data.js` 執行時會印出「點字碼對照表：共 N 筆」，如果這個數字沒變，代表沒有新單字要翻譯，可以直接跳到 commit。

## 情境二：新增了全新的單字（沒出現在點字碼對照表裡）

完整跑一次全部步驟：

```bash
cd E:\Project\en-voc

# 1. 先把新/改過的來源 md 轉成 txt（此時新單字點字碼會是空的）
node en-test/scripts/build-data.js

# 2. 收集全部不重複的英文單字
node en-test/scripts/gen-words-list.js

# 3. 起點字翻譯用的路由伺服器（背景執行，這個視窗留著）
node en-test/scripts/router-server.js &

# 4. 跑批次點字翻譯（需要 Playwright + 本機 Chrome，見下方前提）
node en-test/scripts/braille-translate-batch.js

# 5. 關掉路由伺服器
#    Windows: 用 netstat -ano | grep 8793 找 PID，taskkill //F //PID <pid>
#    或直接 pkill -f router-server.js

# 6. 重新跑一次 build-data.js，這次會讀到新的點字碼表
node en-test/scripts/build-data.js

# 7. 把國小階層資料回填進 index.html
node en-test/scripts/inject-elementary-data.js
```

跑完之後務必**用瀏覽器實際測試**（不能只看 script 有沒有報錯）：

```bash
node server.js &          # 根目錄的本機靜態伺服器，預設 http://localhost:8791/
# 開瀏覽器手動測，或用 Playwright 打開頁面，選幾個課次/冊次確認單字、中文、點字都正常
```

確認沒問題後關閉伺服器，`git add -A && git commit && git push`，GitHub Pages 會自動重新部署（有 CDN 快取，新內容可能要等幾分鐘才會在 `https://hurthuang.github.io/en-voc/` 生效，用 `?v=時間戳記` 這種 cache-busting query string 可以馬上驗證是否真的部署成功）。

## 前提環境

- **Node.js**（純跑 `build-data.js`／`gen-words-list.js`／`inject-elementary-data.js` 只需要這個，沒有額外套件依賴）
- **Playwright**：只有跑 `braille-translate-batch.js` 才需要。如果沒有全域安裝，腳本會試著從 npx 快取路徑找（路徑寫死在腳本開頭的 `resolvePlaywright()`，如果找不到、或機器換過，要改成 `npx playwright --version` 之後實際的 `node_modules/playwright` 路徑，或直接 `npm install playwright` 到專案裡再把 require 改乾淨）。
- **本機 Chrome**：`braille-translate-batch.js` 用 `C:\Program Files\Google\Chrome\Application\chrome.exe`，路徑不同要改腳本裡的 `CHROME_PATH`。
- **E:\Project\tool**：另一個獨立專案，裡面是 liblouis 的 WASM build（`build-no-tables-utf32.js`）跟點字表格（`table/en-ueb-g2.ctb` 等）。`router-server.js` 會讀這個路徑，如果這個工具專案搬過家，要改 `router-server.js` 開頭的 `TOOL_ROOT`。

## 資料格式規則（給人工編輯 md 來源時參考）

- **課次/單元**：`- **課次標籤**：英文1 中文1／英文2 中文2／...`，用全形「／」分隔每個單字項目。
- **詞性**：直接接在英文後面、中文前面，例如 `family n. 家人；家庭`；複合詞性用 `;` 或 `/` 分隔（如 `n.;v.`、`v./n.`），會被完整當作詞性標記保留在中文欄位開頭。
- **點字欄位**：完全不用人工填，永遠由點字批次翻譯自動產生。
- **一個字有多種形態時**（現在式/過去式、單複數、現在分詞、同義替代寫法），**拆成兩個獨立項目**，不要用「/」把兩個字黏在同一個英文欄位裡（這樣測驗時才不會要求使用者打出含斜線的整串字）：
  - 過去式：`began 開始（過去式）`（跟現在式 `begin 開始` 分開兩筆，中文意思相同、過去式後面加註記）
  - 單複數：`leg 腿（單數）／legs 腿（複數）`
  - 現在式/現在分詞：`cook 煮／cooking 正在煮`（中文意思本來就不同，不用額外加註記）
  - 現在式跟過去式完全同形（如 cut/cut、cost/cost）：**只留一筆**，不要重複兩筆一樣的英文
  - 跟時態/單複數無關的同義替代寫法（如 a/an、by metro/MRT）：拆成兩筆，共用同一個中文意思
- **括號注意事項**：英文欄位裡的括號會被 app 在載入時整個拿掉（`grandfather(grandpa)` 顯示/作答時只會是 `grandfather`），所以括號只能拿來寫「替代拼法」，不能拿來寫別的資訊。中文欄位的括號**不會**被拿掉，會完整顯示給使用者看，可以用來寫「（過去式）」「（單數）」這類註記。

## 已知會影響「重跑一次拿到最新資料」的出版社平台細節

這些是蒐集資料時發現的、跟「改版後要不要重新蒐集」有關的重點，完整脈絡見 [資料來源與蒐集方法.md](資料來源與蒐集方法.md) 和各 `英文單字表_*.md` 檔案開頭的說明：

- **康軒 Wonder World／Follow Me**：奇數冊跟偶數冊會出現在不同的「學年度」下拉選單裡（例如 115上看得到奇數冊，其餘學期看得到偶數冊），要切換學年度才會看到完整冊次。
- **何嘉仁 Super Fun**：跟康軒同樣的情況，奇數冊（1/3/5/7）要選「114上」、偶數冊（2/4/6/8）要選「114下」才會出現在冊別選單。第9/10冊確認沒有字彙表資源。
- **翰林 HereWeGo／Dino**：帳號可能同時擁有不同學年度的存取權（例如國中帳號同時有115上跟114下），對應到英語1/3/5跟英語2/4/6，要分別切換學年度下拉選單。
- 出版社改版通常意味著「單字表內容變了」，但**存取路徑（要點哪個下拉選單、哪個檔案分類）不一定會變**，建議先照原本記錄的路徑試試看，行不通再重新摸索。

## build-data.js 的解析邏輯備忘

（給要維護/修改 `en-test/scripts/build-data.js` 的人看，完整程式碼註解更詳細，這裡只列結構）

- `parseJuniorMd()`：解析康軒/翰林/南一國中，靠標題裡的「X年級⋯上/下學期」文字取年級/學期，`Lesson N`/`Unit N` 取課次。
- `parseKangxuanElementary()` / `parseHessElementary()`：`## 系列` → `### 冊次（代號）` → bullet 是課次，兩家出版社結構完全一樣，共用同一套邏輯。
- `parseHanlinElementary()`：跟上面類似，但 HereWeGo/Dino 各自有一點小差異（Dino 有應用/認識/音韻分類需要先清理）。
- `buildBrailleLookup()`：優先讀 `braille-ueb-g2.generated.json`（liblouis 權威表），找不到才退回讀舊 txt 檔殘留的點字碼當備援。
- 點字批次翻譯（`braille-batch.html`）內建一個修正：liblouis 的 en-ueb-g2.ctb 表格裡，it/you/so/do/as 等單格縮寫字緊貼在句尾標點符號時會失效退化成逐字母拼寫，這裡有做繞過（先插入空白讓規則生效，翻譯完再拿掉），詳見該檔案裡的註解。
